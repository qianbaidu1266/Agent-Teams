# 俄罗斯方块（Tetris）页面游戏 — 设计说明

## 1. 页面结构与布局

### 1.1 整体布局

```
┌──────────────────────────────────────────────────┐
│  🎮 俄罗斯方块           🏆 最高分: 0  🎯 0     │ ← 顶栏（标题 + 分数）
├──────────────────┬──────────┬────────────────────┤
│                  │          │                    │
│   [Hold]         │          │   [Next]           │
│   暂存方块       │  游戏    │   下一个方块       │
│   ┌────┐         │  主区域  │   ┌────┐           │
│   │  □ │         │ 10×20   │   │ □□ │           │
│   │    │         │  网格    │   │ □□ │           │
│   └────┘         │          │   └────┘           │
│                  │          │                    │
│  [Ghost] 已开启  │          │   Level: 1         │
│                  │          │   Lines: 0         │
│                  │          │                    │
├──────────────────┴──────────┴────────────────────┤
│  ← → ↓ 移动  ↑ 旋转  Space 硬降  C 暂存  P 暂停 │ ← 操作提示栏
├──────────────────────────────────────────────────┤
│  [▶ 开始]  [⏸ 暂停]  [🔄 重开]                  │ ← 底部控制栏
└──────────────────────────────────────────────────┘
```

### 1.2 各区域尺寸（基于 1080p 参考，按比例缩放）

| 区域 | 尺寸 | 说明 |
|------|------|------|
| 游戏主网格 | 300×600px (10列×20行) | 每格 30×30px |
| 左侧信息栏 | 120px 宽 | Hold 暂存区、Ghost 开关 |
| 右侧信息栏 | 120px 宽 | Next 预览、Level、Lines |
| 顶栏 | 540px 宽 × 48px 高 | 标题 + 分数 |
| 底部控制栏 | 540px 宽 × 42px 高 | 按钮组 |

| **整体容器** | **540×718px** | 包含所有区域 |

### 1.3 适配规则

- 容器居中显示，body 使用 flexbox 居中
- 移动端：整体等比缩放，保留核心交互（触屏按键替换键盘提示）
- 最小宽度 320px，使用 viewport 单位适配

## 2. 用户路径与游戏状态机

### 2.1 状态图

```
┌──────────┐   点击"开始"    ┌──────────┐
│  MENU    │ ──────────────→ │ PLAYING  │
│ (初始态) │                 │ (游戏进行)│
└──────────┘                 └────┬─────┘
      ↑                           │
      │  点击"重开"               │ 按 P / 点击"暂停"
      │                           ↓
      │                    ┌──────────┐
      │                    │ PAUSED   │
      │                    │ (暂停态)  │
      │                    └────┬─────┘
      │                         │
      │                         │ 按 P / 点击"继续"
      │                         ↓
      │                    ┌──────────┐
      │                    │ PLAYING  │
      │                    └────┬─────┘
      │                         │
      │                         │ 方块堆叠触顶
      │                         ↓
      │                    ┌──────────┐
      └─────────────────── │ GAME_OVER│
                           │ (游戏结束)│
                           └──────────┘
```

### 2.2 状态详细定义

| 状态 | 触发条件 | 界面表现 |
|------|----------|----------|
| **MENU** | 页面加载 / "重开"后 | 显示标题动画，主网格半透明显示示例方块或"按开始"提示；"开始"按钮高亮 |
| **PLAYING** | 点击"开始" / 暂停后继续 | 方块正常下落，计时器运行，所有键盘操作生效 |
| **PAUSED** | 按 P / 点击"暂停" | 网格覆盖半透明遮罩，中央显示"⏸ 已暂停"；方块停止下落，时钟冻结 |
| **GAME_OVER** | 新方块无法放入网格 | 网格覆盖半透明遮罩，中央显示"💀 游戏结束"；显示本局得分、最高分；"重开"按钮高亮 |

### 2.3 用户操作映射

| 操作 | 键盘 | 触屏/点击 | 行为 |
|------|------|-----------|------|
| 左移 | ← / A | 左滑 / 左箭头按钮 | 当前方块左移 1 格，碰壁或碰已有方块则无效 |
| 右移 | → / D | 右滑 / 右箭头按钮 | 当前方块右移 1 格 |
| 软降 | ↓ / S | 下滑 / 下箭头按钮 | 方块下移 1 格，触底锁定 |
| 硬降 | Space | 向下长滑 / 硬降按钮 | 方块直接落到底部并锁定 |
| 顺时针旋转 | ↑ / W / X | 点击旋转按钮 | 方块顺时针旋转 90°，踢墙检测 |
| 逆时针旋转 | Z | — | 方块逆时针旋转 90°，踢墙检测 |
| 暂存 | C | 暂存按钮 | 当前方块与 Hold 区互换，每回合仅限一次 |
| 暂停 | P / Escape | 暂停按钮 | 切换暂停/继续 |
| 重开 | — | 重开按钮 | 重置所有状态回到 MENU 或直接新游戏 |

## 3. 核心交互与状态变化

### 3.1 方块（Tetromino）定义

使用标准 7 种方块，每种定义一个 4×4 矩阵，包含所有旋转态（SRS 标准旋转系统）。

| 名称 | 形状 | 颜色 (HEX) | 4×4 矩阵示例 (O 旋转) |
|------|------|------------|----------------------|
| I | ──── | #00f0f0 (青) | 4×4，旋转中心偏移 |
| O | □□<br>□□ | #f0f000 (黄) | 2×2，无旋转变化 |
| T | ┬ | #a000f0 (紫) | 3×3 |
| S | ／／ | #00f000 (绿) | 3×3 |
| Z | ＼＼ | #f00000 (红) | 3×3 |
| J | ┐ | #0000f0 (蓝) | 3×3 |
| L | └ | #f0a000 (橙) | 3×3 |

**SRS 踢墙数据 (Wall Kick Data)**：所有方块（除 O 外）在旋转碰撞时，尝试 5 组偏移量（JLSTZ 一组，I 一组），若某一偏移不碰撞则旋转成功。

### 3.2 游戏循环（tick）

- **基本循环间隔**：从 1000ms（Level 1）开始，每升一级减少 50ms，最低 100ms（Level 18+）
- **每 tick 行为**：
  1. 当前方块下落 1 行
  2. 若无法下落 → 锁定方块到网格 → 检测消行 → 生成新方块
  3. 若新方块与已有格子重叠 → GAME_OVER
- **DAS（按键重复延迟）**：首次按下方向键 170ms 后开始自动重复，之后每 50ms 重复一次

### 3.3 消行与计分

| 同时消行数 | 名称 | 分数 | 分数加成说明 |
|------------|------|------|-------------|
| 1 | Single | 100 × Level | — |
| 2 | Double | 300 × Level | — |
| 3 | Triple | 500 × Level | — |
| 4 | Tetris | 800 × Level | 四行同时消除 |
| 软降 | — | 每格 1 分 | — |
| 硬降 | — | 每格 2 分 | — |

### 3.4 等级与速度

| 条件 | 行为 |
|------|------|
| 每消除 10 行 | Level += 1 |
| Level 每升 1 | 下落间隔 = max(100, 1000 - (Level - 1) × 50) ms |
| 最高 Level | 18+（间隔 100ms 封顶） |

### 3.5 Ghost Piece（影子方块）

- 定义：从当前方块位置垂直向下投影到最底落点的半透明轮廓
- 视觉：与当前方块同色，透明度 20%，边框线保留
- 更新时机：每次方块移动或旋转后立即重新计算
- 开关：默认开启，可通过 Ghost 按钮切换

### 3.6 Hold（暂存区）

- 每回合（从生成到锁定）仅能使用 1 次 Hold
- 行为：
  - Hold 区为空 → 当前方块存入，生成下一个方块
  - Hold 区非空 → 交换当前方块与 Hold 区方块，重置位置到初始
- 视觉：Hold 区显示下一个方块的预览（颜色 + 形状，缩小到 20×20px/格）
- 状态指示：本轮已使用 Hold 时，Hold 按钮或 Hold 区域变灰禁用

### 3.7 Next（下一个方块预览）

- 显示即将出现的下一个方块（颜色 + 形状）
- 使用 4×4 网格，每格 20×20px，居中绘制
- 更新时机：当前方块锁定后，Next 方块变为当前方块，生成新 Next

## 4. 视觉方向

### 4.1 整体风格

- **主题**：深色霓虹/赛博朋克风
- **背景**：深蓝紫色渐变 `linear-gradient(145deg, #0f0c29, #1a1a2e, #16213e)`
- **容器**：半透明白色圆角卡片，带毛玻璃效果（backdrop-filter: blur）
- **字体**：系统无衬线字体（Segoe UI, system-ui），数字用等宽字体

### 4.2 游戏网格

| 元素 | 样式 |
|------|------|
| 网格背景 | #0b1a32, 内阴影深色 |
| 网格线 | 1px rgba(255,255,255,0.03) 细线（可选，建议不画线，用格间距表现） |
| 已填充方块 | 对应颜色 + 渐变为上方略亮、下方略暗，形成立体感 |
| 方块边框 | 内侧高光线（左上 1px rgba(255,255,255,0.25)），右下 1px rgba(0,0,0,0.3) |
| 幽灵方块 | 同色 20% 透明度 + 虚线边框 |
| 当前方块高亮 | 轻微发光效果（filter: drop-shadow） |

### 4.3 颜色系统

```
— I: #00f0f0  → 主色 #00f0f0, 高光 #66ffff, 阴影 #009999
— O: #f0f000  → 主色 #f0f000, 高光 #ffff66, 阴影 #999900
— T: #a000f0  → 主色 #a000f0, 高光 #cc66ff, 阴影 #660099
— S: #00f000  → 主色 #00f000, 高光 #66ff66, 阴影 #009900
— Z: #f00000  → 主色 #f00000, 高光 #ff6666, 阴影 #990000
— J: #0000f0  → 主色 #0000f0, 高光 #6666ff, 阴影 #000099
— L: #f0a000  → 主色 #f0a000, 高光 #ffcc66, 阴影 #996600
— UI文字: #e0e0e0
— 强调色: #e94560 (按钮/分数)
— 金色: #ffd700 (最高分)
```

### 4.4 动画与过渡

| 场景 | 动画 |
|------|------|
| 方块下落 | 平滑的每格步进（不插值，符合格线逻辑） |
| 消行 | 消行行闪烁白色 → 收缩消失（200ms）→ 上方行下落 |
| 方块锁定 | 轻微闪光效果 |
| 游戏结束 | 网格渐变覆盖 + 文字淡入 |
| 暂停 | 网格快速模糊（CSS blur） |

## 5. 工程实现建议（给工程师）

### 5.1 技术选型建议

- 纯 HTML + CSS + JavaScript（单页，无需框架）
- Canvas 2D 渲染游戏主网格
- DOM 元素渲染 UI（分数、按钮、Next/Hold 预览区用 div 或小 Canvas）
- 或全部 Canvas 渲染（更统一，但 DOM 方式更易维护 UI 交互）

### 5.2 必须实现的行为清单（最低验收标准）

1. **7 种标准方块**完整定义，包括 I 和 O 的特殊旋转逻辑
2. **SRS 踢墙系统**：JLSTZ 和 I 各自的 5 组踢墙偏移
3. **时钟驱动下落**：基于 Level 的动态间隔，requestAnimationFrame + 时间累加
4. **锁定延迟**：方块触底后 500ms 锁定，期间可移动/旋转
5. **消行动画 + 计分 + 等级提升**
6. **Ghost piece** 实时更新
7. **Hold** 功能，每回合一次限制
8. **Next** 预览
9. **暂停/继续**，状态切换正确
10. **游戏结束** 判定与重置
11. **键盘控制**（方向键 + WASD + Space + C + P）
12. **触屏友好**：提供屏幕按钮（上下左右旋转暂停），防止页面滚动
13. **最高分** localStorage 持久化

### 5.3 数据模型参考结构

```javascript
// 核心数据结构
const game = {
  board: [],           // 20×10 二维数组，0=空，1-7=对应方块色索引
  currentPiece: {      // 当前活动方块
    type: 'T',         // I/O/T/S/Z/J/L
    rotation: 0,       // 0-3
    x: 3,              // 网格列偏移
    y: 0               // 网格行偏移
  },
  nextPiece: null,     // 下一个方块
  holdPiece: null,     // 暂存方块
  canHold: true,       // 本回合是否可用 Hold
  score: 0,
  highScore: 0,
  level: 1,
  lines: 0,
  state: 'MENU',       // MENU | PLAYING | PAUSED | GAME_OVER
  dropInterval: 1000,  // ms
  lastDrop: 0,         // 时间戳
  lockTimer: null,     // 锁定延迟计时器
};
```

## 6. 异常状态与边界处理

| 场景 | 处理方式 |
|------|----------|
| 浏览器窗口 resize | 按比例缩放游戏容器，保持宽高比，不超过视口 |
| localStorage 不可用 | 静默降级，最高分仅内存记录，不报错 |
| 触屏设备同时多点触控 | 仅响应第一个触摸点，防止误操作 |
| 页面在游戏进行中切换标签页 | 利用 Page Visibility API 自动暂停游戏 |
| 方块生成时与已有格子重叠 | 判定 GAME_OVER，不清除已有格子 |
| 空 Hold 区执行交换 | 当前方块放入 Hold，生成新 Next 作为当前 |
| 旋转踢墙全部失败 | 保持原位置和旋转角度，不做任何变化 |
| 连续快速按键 | 按按键队列顺序处理，每帧最多处理一次移动/旋转 |
